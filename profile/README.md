## 📖 Overview

<a href="#">GraphPad Prism</a> combines scientific graphing and statistical analysis in a single application designed specifically for biomedical researchers, not for statisticians. The software assumes you know your experimental design and need to analyze and present results without learning a general-purpose statistical programming environment.

The <a href="#">data organization system</a> uses structured table formats matched to specific analysis types — XY tables for dose-response and time-course data, grouped tables for comparing multiple treatment groups, contingency tables for categorical data analysis — guiding data entry toward the correct format for the intended analysis. <a href="#">Guided analysis workflows</a> prompt for experimental design parameters before running tests, reducing the likelihood of applying an incorrect statistical test to a dataset by walking through the relevant experimental considerations before analysis begins.

<p align="center">
  <img src="https://www.graphpad.com/guides/prism/latest/user-guide/images/clip0024.png" alt="GraphPad Prism screenshot"/>
</p>

<a href="#">Nonlinear curve fitting</a> fits data to over 100 built-in equation models — dose-response curves, enzyme kinetics, binding saturation, pharmacokinetic models — with automatic initial parameter estimation, confidence intervals for fitted parameters, and goodness-of-fit statistics. Custom equations allow fitting to models not in the built-in library. <a href="#">Publication-quality graph output</a> formats graphs according to journal specifications with precise control over every visual element — axis labels, tick marks, error bar styles, symbol types, and color schemes — without requiring graphics software post-processing.

The <a href="#">multiple comparisons testing</a> applies appropriate post-hoc corrections for one-way and two-way ANOVA, with clear visual indication of which group comparisons reach statistical significance and customizable significance threshold annotation on graphs. <a href="#">Survival analysis</a> generates Kaplan-Meier curves, log-rank tests, and hazard ratios from censored time-to-event data in the format that clinical and preclinical researchers require.

GraphPad Prism's dominance in biomedical research is a product of the specific gap it fills: between spreadsheet software that can graph but cannot run sophisticated statistics, and statistical programming environments that can run any analysis but require programming fluency most experimental scientists do not have. Prism sits precisely in the middle — capable enough for the full range of standard biomedical analyses, accessible enough for scientists whose primary expertise is biology or pharmacology rather than statistics. That positioning has made it the default choice across a research community that largely standardized on it through graduate training and peer lab adoption.

## 🤔 FAQ

<details>
<summary>Is GraphPad Prism suitable for non-biomedical research?</summary>

Yes. While designed with biomedical research in mind, Prism handles general scientific data analysis including curve fitting, statistical comparison, and graphing for any quantitative research discipline.

</details>

<details>
<summary>Can Prism handle large datasets?</summary>

Prism works well for typical experimental datasets. Very large datasets with thousands of rows may be better suited to dedicated statistical software. Prism is optimized for experimental research data rather than large-scale data science.

</details>

<details>
<summary>Does Prism guide you in choosing the correct statistical test?</summary>

Yes. Guided analysis workflows ask about experimental design — number of groups, whether data are paired, distribution assumptions — before presenting appropriate test options with explanations of when each applies.

</details>

<details>
<summary>What graph types does Prism support?</summary>

XY scatter, bar, column, box-and-whisker, violin, survival, heat map, bubble, and many others. Graphs link directly to analyzed data and update automatically when data change.

</details>
